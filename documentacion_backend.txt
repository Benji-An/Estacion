=====================================================
    DOCUMENTACIÓN DEL BACKEND (FastAPI)
=====================================================

1. ¿CÓMO PONER A FUNCIONAR EL PROYECTO?
-----------------------------------------------------
Paso 1: Activar el entorno virtual
En Windows: `.\.venv\Scripts\activate`

Paso 2: Instalar dependencias (solo si es la primera vez o añadiste algo nuevo)
`pip install -r requirements.txt`

Paso 3: Levantar el servidor
`uvicorn app.main:app --reload`
*Nota: El servidor estará disponible en http://localhost:8000*
*La documentación automática de la API (Swagger) está en http://localhost:8000/docs*

Paso 4 (Opcional): Aplicar migraciones a la Base de Datos
Si se han modificado los modelos:
`alembic revision --autogenerate -m "mensaje"`
`alembic upgrade head`


2. VARIABLES DE ENTORNO Y CONFIGURACIÓN
-----------------------------------------------------
* ¿Dónde se definen las variables? 
  En el archivo `.env` en la raíz del proyecto (Ej: DATABASE_URL, SECRET_KEY).
* ¿Cómo las lee Python?
  En el archivo `app/core/config.py`. Allí se usa Pydantic para leer el `.env` y cargar las variables en un objeto llamado `settings`.
* ¿Cómo se usan en el resto del código?
  Simplemente importas `settings` donde lo necesites.
  Ejemplo: `from app.core.config import settings`
  Y luego usas: `settings.DATABASE_URL`


3. ESTRUCTURA DEL PROYECTO (¿DÓNDE ESTÁ CADA COSA?)
-----------------------------------------------------
El código principal vive en la carpeta `app/`.

* app/main.py: 
  Es el punto de entrada. Aquí se inicializa FastAPI y se conectan todos los endpoints.

* app/api/ (Controladores / Endpoints):
  Aquí están las rutas a las que el frontend puede llamar.
  - `app/api/v1/endpoints/auth.py`: Rutas `/login/access-token` y `/register`.
  - `app/api/v1/endpoints/upload.py`: Ruta `/upload` para recibir el archivo CSV.

* app/models/ (Base de Datos):
  Aquí defines cómo son tus tablas en PostgreSQL.
  - `user.py`: Define la clase `User` (email, contraseña cifrada, id).

* app/schemas/ (Validadores Pydantic):
  Definen cómo deben ser los datos que entran (Requests) y salen (Responses) de la API.
  - `user.py`: Define qué campos se piden para crear un usuario y cuáles se devuelven.
  - `token.py`: Define la estructura del token de acceso devuelto.

* app/crud/ (Operaciones de Base de Datos):
  CRUD = Create, Read, Update, Delete. 
  Aquí están las funciones que hablan directamente con SQLAlchemy para buscar, guardar o borrar datos.
  - `user.py`: Tiene funciones como `get_by_email` o `create` para guardar un usuario en la BD.

* app/services/ (Lógica de Negocio):
  Aquí va el código que hace cálculos o procesamientos pesados, para no saturar los endpoints.
  - `csv_service.py`: Contiene la lógica para procesar archivos.

* app/core/ y app/db/:
  Configuraciones centrales, seguridad (cifrar contraseñas, generar tokens JWT) y la conexión a la base de datos (`session.py`).


4. ¿QUÉ HACEN LAS FUNCIONES PRINCIPALES?
-----------------------------------------------------
A) El Flujo de Registro (app/api/v1/endpoints/auth.py -> register_user):
  1. Recibe el email y contraseña del frontend.
  2. Llama a `crud_user.get_by_email` para verificar si el email ya existe. Si existe, lanza un error 400.
  3. Llama a `crud_user.create` que se encarga de cifrar (hashear) la contraseña y guardar al usuario en la BD.

B) El Flujo de Login (app/api/v1/endpoints/auth.py -> login_access_token):
  1. Recibe username (email) y password en formato formulario.
  2. Busca al usuario en la BD.
  3. Usa la función `security.verify_password` para comprobar si la contraseña enviada coincide con el hash guardado.
  4. Si todo está bien, usa `security.create_access_token` para generar un JWT y se lo envía al frontend.

C) El Flujo de Carga de CSV (app/api/v1/endpoints/upload.py -> upload_csv):
  1. Recibe el archivo enviado por el frontend mediante un form (`multipart/form-data`).
  2. Extrae el contenido del archivo de la memoria (en texto utf-8).
  3. Llama a la función `parse_csv_rows` del archivo `app/services/csv_service.py`.

D) Función de Servicio de CSV (app/services/csv_service.py -> parse_csv_rows):
  1. Recibe el texto bruto del CSV.
  2. Analiza los primeros caracteres para intentar "adivinar" si está separado por comas `,`, punto y coma `;` u otro.
  3. Extrae las cabeceras (headers) de las columnas.
  4. Convierte las filas del CSV en una lista de diccionarios JSON (ej: `{"Nombre": "Juan", "Edad": "30"}`).
  5. Retorna esta lista de vuelta al endpoint, quien se encarga de enviarla al frontend.
